RagnarWM

Minimal, flexibal & userfriendly.

An Overview

Ragnar is a dynamic tiling window manager for X written entirely in C. Codewise, the project is very minimal to avoid bugs and messy things. Ragnar uses Xlib for it's X-Server interface. Currently (v1.2) the project has about 2.1k lines of code.

Installation

There are some installation scripts for a few distros like ubuntu & arch. Those make the process of installing very easy & automated.

The Arch Script

Installing Ragnar with the arch-installation script is as easy as:

cd ~/ sh -c "$(curl -fsSL https://raw.githubusercontent.com/cococry/Ragnar/main/install/arch_installation.sh)"

The Ubuntu Script

Just run the script on your Ubuntu mashine and you're good to go:

cd ~/ sh -c "$(curl -fsSL https://raw.githubusercontent.com/suleyman-kaya/Ragnar/main/ubuntu_installation.sh)"

Installation without a Script

Installing Ragnar without a script is also a fairly easy process. Just follow these easy steps:

Installing Dependencies

Ragnar requires a few dependencies to be installed on your system. Those are:

  • xorg
  • xorg-xinit
  • git
  • gcc
  • make
  • sudo

On Arch

Install these packages with pacman:

sudo pacman -S xorg xorg-xinit git gcc make sudo

On Debian

The dependencies are installed with apt:

sudo apt-get install xorg xorg-xinit git gcc make sudo

After installing the build dependencies to your system, the next step is to clone the Ragnar git repository

Clone the Git-Repository

To clone the repository run:

git clone --depth=1 https://github.com/cococry/Ragnar

If the repository folder is on your system, cd into it and install ragnar to your mashine with the 'make' command.

Installation with 'make'

Installing ragnar onto your mashine is as easy as:

sudo make ragnar install

This requires the make package being installed and also access to sudo. The Makefile compiles ragnar and copies it to your /usr/bin directory.

Setting up .xinitrc

Now you need to specify to use Ragnar as your X Window manager. On startup of the X-Server, ragnar should start.

To do this you need to create a .xinitrc file in your home directory:

cd ~/ touch .xinitrc

Now, add 'exec ragnar' to your .xinitrc to start ragnar on X-Startup:

echo exec ragnar >> .xinitrc

Starting Ragnar on boot

If you want to start Ragnar as your PC boots, you need to call the 'startx' command which executes .xinitrc which does 'exec ragnar'.

To do this, add an if statement that checks if the X-Server has been started yet to your shell's rc file (.zshrc, .bashrc etc.):

if [[ ! $DISPLAY && $XDG_VTNR -eq 1 ]]; then startx fi

Adding Ragnar to your sessions

If you are using a session manager you need to add Ragnar to your session, so you can use it.

To do that, copy the ragnar.desktop file in your Ragnar directory to /usr/share/xsessions:

sudo cp ragnar.desktop /usr/share/xsessions/ragnar.desktop

Conclusion

That's it, you installed Ragnar to your system and now using it as your window manager. For further explanations, tutorials & information, there are lots of them on this site. If you're having any trouble installing ragnar or general problems, feel free to sumbit an issue on GitHub.

Tutorial

Booting into ragnar the first time can be pretty overwhelming for new users. How can i open a terminal? How can i move windows? These questions and more are going to be answered in this chapter. After reading this you will be able to use Ragnar like a pro.

Opening keybinded apps

In Ragnar, by default there are keybinds for opening your terminal, your browser & your application launcher. Those are:

Terminal

Super + Enter

The default terminal of Ragnar is alacritty. If nothing happens when you press Super + Enter, install alacritty to your system and it will work.

Browser

Super + W

Ragnar uses the chromium browser as it's default browser. If nothing happens when you press Super + W, install the 'chromium' package to your system and it will work.

Application Launcher

Super + S

Dmenu is used as the default application launcher of ragnar. If pressing Super + S does nothing, install the 'dmenu' package to your system and it will work.

Working with Windows

Now, that you can open basic windows like a terminal, you may ask yourself how you resize a window or move one. This and more is going to be explained in this chapter.

Selecting windows

Selecting a window is as easy as hovering over it with your mouse. This will change it's border color to 'WINDOW_BORDER_COLOR_ACTIVE' which is the border-color of selected windows in the config file. It also sets the X input-focus to the window you are hovering. This method does not raise the window.

It is possible to off hovering over windows to select them in the config file. You can read more about configuring in the configuration chapter.

You can also select a window by cycling to it. This is done by pressing:

Super + Tab

This cycles through all of the windows that are on your selected monitor and on your selected desktop. The window will also be highlighted by the 'WINDOW_BORDER_COLOR_ACTIVE' color and the X input focus will be changed to the window. This also raises the window.

The last method of selecting a window in Ragnar is by hard-selecting it. This is done by pressing:

Super + Middle Mouse

Hard-selecting a window will cause it to be marked with the 'WINDOW_BORDER_COLOR_HARD_SELECTED' color. If any window is hard selected, hovering over other windows will be ignored. Hard selection can be toggled by pressing the Super key and the key specified for hard-selecting in the config file.

Moving windows

To move windows around, hold and press:

Super + Left Mouse

Resizing windows

To change the size of windows, hold and press:

Super + Right Mouse

Window Layout

Ragnar, as it is a tiling window manager, it has tiled layouts. By default there are four window layouts, officially labled: Tiled Master, Horizontal Master, Vertical Stripes & Horizontal Stripes. A floating mode is also available. You can dynamically change the size of the master and the slave windows in the layout, alter the gap in the layout, move windows around and alter the master window. All of that functionality will be explained in this section.

The default window layout in the default configuration is set to the Tiled Master layout. To the left, you have one master window and to the right there is a stack of slave windows.

Altering the layout

Changing slave size

To change the change (x or y, depending on layout) of a slave window, use:

Super + J

Super + K

Note that you can only change the width or height of a window in a layout that way as long it is not lower than WINDOW_MIN_SIZE_LAYOUT_HORIZONTAL or WINDOW_MIN_SIZE_VERTICAL. Also, changing a slave window's size that way only works if there are two or more slaves on the stack. Changing slave size only works in layouts where a concept of slaves and masters exists.

Changing master size

To change size size of the master window in the layout, use:

Super + H

Super + L
